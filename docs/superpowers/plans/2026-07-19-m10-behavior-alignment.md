# M10 — 行为对齐（基于官方 demo 实测差异清单）

依据：2026-07-19 双代理真实浏览器实测官方 editor-starter demo vs 本地，41 项差异（9 must-fix）。

## T1：时间线拖拽模型重做（TimelinePanel.tsx, ops.ts）

官方模型（实测确认）：
- 拖拽中块本体 1:1 跟随光标（双轴，渲染在最上层，可越界到标尺/轨道头上方）
- 深灰圆角**落位槽**实时显示合法落点：负帧钳到 0；与占位块重叠时紧贴其后；悬停轨道间隙显示细灰插入条；悬停最上/最下方实时插入新空轨道行（轨道号实时重排）
- 松手落到槽位置，**永不回弹**；空轨道拖完自动移除
- 拖拽中吸附到其他块边缘/播放头，出现贯穿整个时间线高度的竖直吸附线
- 修剪也吸附（~3px 阈值）并显示吸附线
- 视口左右边缘悬停自动滚动
- 多选拖动只移动被抓的块
- 总时长/标尺在松手后才更新（拖拽中不改 store，天然满足）

## T2：时间线表面与菜单（ItemBlock.tsx, Playhead/Ruler, TrackHeader, import 落位）

- 块右键菜单（shadcn ContextMenu）：剪切 / 复制 / 创建副本 ┃ 置于顶层 / 置于底层（右键即选中）
- 选中样式：1px 蓝边框 #0B84F3（替代厚白框）；块体 pointer；修剪手柄 e-resize/w-resize
- 淡变手柄：顶角白色小药丸（悬停显示），拖动带「淡入 0.8s」提示与深色楔形
- 音量线：1px 白 20% 透明度、线下灰色填充、拖动显示 dB 提示
- 视频块左上角显示文件名
- 播放头改蓝色带标尺内把手；标尺按住拖动连续 scrub
- 轨道头只显编号、自下而上编号（按位置实时计算，顺带修掉 paste+undo 后名字重复的 bug）
- 导入落位：落在播放头处有空间的现有轨道，而非总是新建顶轨
- 时间线缩放加「适应」按钮（官方 zoom 0 = fit 的务实近似）

## C：画布对齐（canvas/**, App.tsx, add-items.ts, useShortcuts.ts）

- 右键菜单补全：剪切 / 复制 / 创建副本 ┃ 置于顶层 / 置于底层；空白处右键无菜单
- 框选：空白处拖出 #0B84F3 边框 + 10% 填充矩形，touched 即选中，拖拽中实时预高亮
- 手柄改官方样式：4 角白色方块（蓝边）+ 边缘全长隐形热区（正确 ns/ew 光标）；
  角拖 solid/text 默认自由缩放，媒体默认锁比例 + Shift 解锁
- 悬停未选中项显示 2px 蓝色描边
- 选中项下方蓝色 W×H 尺寸徽章，移动/缩放/绘制中实时更新；拖动中手柄不消失
- 文本工具改「点击放置」：按钮进入模式（I-beam 光标），点击处放置自适应尺寸文本（字号 80），不自动进入编辑
- 绘制色块：拖拽中实时渲染真实色块（默认白色）+ 选中 UI + 尺寸徽章；单击 = 100×100
- 行内编辑时隐藏手柄/徽章，修复底层文字重影
- 缩放快捷键改相对步进（+ 加倍 / − 减半 / 0 适应）；缩放控件移入顶部工具栏（适应图标 + − + 标签 + +）
- 吸附辅助线改品红色

## 刻意不对齐（保留我们的做法）

- 中文界面（产品决策）
- Ctrl/触控板捏合缩放（用户明确要求过，官方反而没有）
- 媒体最大裁剪指示器（官方功能清单有此 flag，demo 未开；保留）
- Esc 退出裁剪模式、多选时检查器显示「已选 N 项」（无害超集）
- 音量增益超 0dB（官方可 +20dB；我们 0..1，dB 增益放大列入后续）
- 媒体轨道行更高、时间线 fit 缩放模型（用「适应」按钮近似）——后续
