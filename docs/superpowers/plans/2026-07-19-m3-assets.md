# M3：素材管线 实施计划

> REQUIRED SUB-SKILL: superpowers:executing-plans

**Goal:** 素材导入（拖放/按钮）→ mediabunny 探测元数据 → 建 Asset+Item → MinIO 预签名上传 → IndexedDB 缓存；shared 渲染器支持 video/image/audio/gif（含 trim/speed/volume/fade/crop/静音轨道）；时间轴胶片缩略图 + 音频波形。验收：拖入视频即可编辑并播放。

**规格来源:** specs §6.1 §5.2、M3 验收

## Global Constraints

- 预览用本地 blob URL 优先（`assetUrlOverrides` 合成 prop，渲染服务不传即用远端 URL）
- `assetStatus`（pending-upload/in-progress/uploaded/error）为瞬时状态，不进撤销栈
- MinIO 走 S3 协议（forcePathStyle），桶匿名可读（开发期）；生产切 OSS 只改 env
- gif 不受素材时长限制（可循环）；video/audio 修剪受限

## Tasks

1. **T1 server + MinIO**：`apps/server`（Fastify：`POST /api/upload` 预签名、`GET /healthz`）+ 根 `docker-compose.yml`（MinIO 9100/9101）+ 启动时建桶/设匿名读策略 + vite `/api` 代理。检验：启动后 curl 拿预签名 URL 并 PUT 成功。
2. **T2 shared 媒体渲染器**：Video/Audio（@remotion/media，trimBefore 换算 = item.trimBefore、playbackRate、volume 回调乘 fade、muted∥轨道静音）、Img（含 crop 公式）、Gif（@remotion/gif，loop）。`MainComposition` 增加可选 `assetUrlOverrides`。ops：gif 的 maxItemDurationInFrames 返回 null。
3. **T3 导入流程**：`lib/probe.ts`（mediabunny：时长/宽高/hasAudio；图片 createImageBitmap）、`lib/import-assets.ts`（建 asset+item：按合成适配缩放、放当前帧、无空位则新建顶部轨道；上传状态机；IndexedDB 缓存 `caching/indexeddb.ts`）、顶栏导入按钮 + 画布拖放。store 增 `assetStatus/localUrls`。
4. **T4 胶片缩略图 + 波形**：`timeline/Filmstrip.tsx`（video seek 抽帧，按 assetId+宽度缓存）、`timeline/Waveform.tsx`（decodeAudioData 峰值绘制，按 assetId 缓存）。
5. **T5 端到端验证**：fixtures 由脚本生成（node 写 PNG/WAV；浏览器 MediaRecorder 录 WebM），Playwright 走完整链路：导入 → 时间轴出条块（视频含缩略图/音频含波形）→ 播放器可见 → MinIO 里对象存在 → 状态 uploaded。

检验通过后逐任务 commit。
