# M9 — shadcn UI 打磨（对齐 editor-starter 成品观感）

现状：功能已对齐官方清单，但 UI 是 emoji + 原生控件的开发风格；shadcn 60 组件已装但未用。
目标：全面换用 shadcn 组件与 lucide 图标，接入 toast 反馈，不改变任何交互行为。

## 块 U1：应用外壳（App.tsx, playback/**, TimelinePanel chrome, SelectionOverlay, lib 反馈）

- 头部工具栏 / 播放条 / 时间线工具栏 / 轨道头：lucide 图标替换 emoji（⏯🧲👁🔇 等），
  shadcn Button + Tooltip；保留既有 title/名称（跳到结尾、循环、清理素材、绘制色块…）。
- 画布右键菜单换 shadcn ContextMenu（保留 置于顶层/置于底层 文案）。
- 清理素材 confirm() 换 AlertDialog。
- sonner toast 接入：导入/上传失败、渲染完成/失败、字幕完成/失败、保存成功。
- 受影响的 e2e 选择器（verify-m4 菜单 role、verify-m5 对话框）同步更新并回归。

## 块 U2：检查器（inspector/**）

- 数字输入保留 NumberField 拖拽行为；range → shadcn Slider；select → shadcn Select
  （字重下拉保留 hover 实时预览）；checkbox → Switch/Checkbox；分区加 Separator。
- 保留名称：交换尺寸、渲染、生成字幕、对齐、裁剪、源信息。
- 受影响的 e2e（verify-m1-inspector / m6 / m7）如 role 变化则同步更新并回归。

## 硬约束

- 不改行为、不动 data-* 钩子（data-item-block/volume-line/fade/trim/tl-scroll、cursor-crosshair）。
- 不新增依赖（lucide-react 已随 shadcn 安装）。
- typecheck + vitest + 全部 verify 脚本绿。
